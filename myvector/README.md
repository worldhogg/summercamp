# MyVector

> Шаблонный динамический контейнер на C++17 — упрощённый аналог `std::vector`

Собственная реализация динамического массива с автоматическим управлением памятью. Контейнер растёт по мере добавления элементов, поддерживает полную семантику копирования и перемещения. Проект демонстрирует понимание того, как устроены контейнеры стандартной библиотеки изнутри.

---

## Что демонстрирует проект

- **Шаблоны (templates)** — контейнер работает с любым типом `T` через `template <typename T>`
- **RAII** — память выделяется в конструкторе и автоматически освобождается в деструкторе
- **Правило пяти (Rule of Five)** — корректное копирование и перемещение объекта, владеющего динамической памятью
- **Move-семантика** — перемещение ресурса вместо копирования, `noexcept`-операции
- **Copy-and-swap идиома** — безопасное присваивание с защитой от самоприсваивания
- **Динамический рост (реаллокация)** — автоматическое расширение массива при заполнении
- **Перегрузка операторов** — `operator[]` для индексного доступа (чтение и запись)

---

## Как устроен контейнер

Внутри — динамический массив элементов типа `T` и два счётчика:

- **size** — количество реально хранимых элементов
- **capacity** — объём выделенной памяти

Разделение `size` и `capacity` позволяет добавлять элементы без выделения памяти на каждый вызов. Когда массив заполняется (`size == capacity`), контейнер выделяет новый массив вдвое большего размера, копирует туда существующие элементы, освобождает старый и продолжает работу. Такой подход обеспечивает быстрое добавление в среднем.

При копировании контейнера выполняется глубокое копирование (deep copy) — каждый объект получает независимый массив, что исключает двойное освобождение памяти. При перемещении ресурс передаётся без копирования.

## Возможности

| Метод | Описание |
|---|---|
| `push_back(value)` | добавить элемент в конец (с реаллокацией при необходимости) |
| `pop_back()` | удалить последний элемент |
| `operator[](i)` | доступ по индексу — чтение и запись (обычная и `const` версии) |
| `front()` / `back()` | первый / последний элемент |
| `clear()` | очистить контейнер |
| `empty()` | проверка на пустоту |
| `getsize()` | количество элементов |
| `getcapacity()` | объём выделенной памяти |

**Правило пяти:** деструктор, копирующий конструктор, копирующий `operator=` (через copy-and-swap), move-конструктор. Move-присваивание обслуживается идиомой copy-and-swap.

## Сборка и запуск

Шаблонный класс реализован целиком в заголовочном файле, поэтому отдельного `.cpp` для контейнера нет — только заголовок и файл с тестами.

```bash
g++ -std=c++17 -Iinclude src/main.cpp -o vec
./vec
```

---

## Пример использования

```cpp
MyVector<int> v;
v.push_back(10);
v.push_back(20);
v.push_back(30);

std::cout << v.front() << " " << v.back() << "\n";   // 10 30
std::cout << v.getsize() << "\n";                    // 3

v[0] = 999;                                          // запись через operator[]
v.pop_back();
std::cout << v.back() << " " << v.getsize() << "\n"; // 20 2

// Глубокое копирование — независимые массивы
MyVector<int> copy = v;
copy[0] = 1;
std::cout << v[0] << " " << copy[0] << "\n";         // 999 1
```
