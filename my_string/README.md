# MyString

Собственная реализация строкового класса на C++17 — упрощённый аналог `std::string`.

Проект создан для практического освоения современного C++:
- **RAII** (Resource Acquisition Is Initialization)
- **Правило пяти** (Rule of Five) — полная реализация
- Ручное управление динамической памятью
- Move-семантика и rvalue-ссылки
- Перегрузка операторов

## Возможности

### Конструкторы и деструктор
- Конструктор по умолчанию (пустая строка)
- Конструктор из C-строки (`const char*`)
- Конструктор копирования (глубокое копирование)
- Конструктор перемещения (передача владения)
- Деструктор (автоматическое освобождение памяти)

### Операторы присваивания
- Копирующее присваивание (copy-and-swap idiom)
- Перемещающее присваивание (`noexcept`)

### Доступ и информация
- `size()` — длина строки
- `empty()` — проверка на пустоту
- `c_str()` — указатель на C-строку (для совместимости с C-API)
- `operator[]` — доступ к символу по индексу (const и не-const версии)

### Манипуляции
- `clear()` — очистка строки
- `operator+` — конкатенация (создаёт новый объект)

### Сравнение
- `operator==`, `operator!=` — посимвольное сравнение

### Ввод-вывод
- `operator<<` — вывод в `std::ostream`

## Пример использования

\`\`\`cpp
#include "MyString.h"
#include <iostream>

int main() {
    // Создание
    MyString hello("Hello, ");
    MyString world("World!");
    
    // Конкатенация
    MyString greeting = hello + world;
    std::cout << greeting << std::endl;  // "Hello, World!"
    
    // Копирование (глубокое — независимые объекты)
    MyString copy = greeting;
    copy[0] = 'h';
    std::cout << "Original: " << greeting << std::endl;  // "Hello, World!"
    std::cout << "Modified: " << copy << std::endl;     // "hello, World!"
    
    // Перемещение (без копирования содержимого)
    MyString moved = std::move(greeting);
    // greeting теперь пустая, moved владеет содержимым
    
    // Сравнение
    MyString a("test"), b("test");
    if (a == b) {
        std::cout << "Equal!" << std::endl;
    }
    
    return 0;
}
\`\`\`

## Что я узнал в процессе

- **RAII** — фундаментальная идиома C++. Ресурс живёт ровно столько, сколько объект, который им владеет. Без RAII невозможно писать безопасный код с динамической памятью.

- **Правило пяти** — если у класса есть нетривиальный деструктор, обязательно нужны все 5 специальных методов (деструктор + 2 копирующих + 2 перемещающих). Иначе будут утечки или двойное освобождение памяти.

- **Copy-and-swap idiom** — безопасный паттерн для оператора присваивания. Сначала создаём копию, потом освобождаем старое — exception-safe.

- **Разница между копированием и перемещением:**
  - Копирование: выделяет новую память + копирует содержимое — O(n)
  - Перемещение: переключает указатель — O(1), но source становится пустым

- **lvalue vs rvalue** — категории выражений, определяющие, какой метод выберет компилятор. `lvalue` — это объект с именем и адресом. `rvalue` — временное значение.

- **`std::move`** не двигает объект. Он только меняет тип выражения на rvalue, разрешая move-операцию.

- **`noexcept` на move-операциях** — обязательно для эффективной работы с STL. Без этого `std::vector` будет копировать вместо перемещения при реаллокации.

- **Отладка памяти** — научился различать ошибки **компилятора** (синтаксис, redefinition) и **линковщика** (undefined symbols, multiple definition). Это разные стадии сборки и лечатся по-разному.

## Возможные улучшения

В рамках учебного проекта **не реализовано** (но можно добавить):
- `operator+=` для модификации строки на месте
- Методы `append()`, `substr()`, `find()`
- `at()` с проверкой границ и исключениями
- Итераторы для совместимости с STL-алгоритмами
- SSO (Small String Optimization) — оптимизация для коротких строк
- Поддержка Unicode (текущая версия работает только с ASCII)
