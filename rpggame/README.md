# Text RPG

> Консольная ролевая игра на C++17 — практика объектно-ориентированного проектирования

Пошаговая боевая система, построенная на иерархии классов и полиморфизме. Проект демонстрирует ключевые принципы ООП в современном C++: наследование, виртуальные функции, абстрактные классы и автоматическое управление памятью через умные указатели.

---

## Что демонстрирует проект

- **Иерархия наследования** в два уровня — абстрактный базовый класс, промежуточный абстрактный класс и конкретные реализации
- **Полиморфизм времени выполнения** — единая боевая функция работает с любыми типами существ через указатели на базовый класс
- **Абстрактные классы и чисто виртуальные функции** — общий интерфейс, который обязаны реализовать наследники
- **Виртуальный деструктор** — корректное освобождение ресурсов при удалении объекта через указатель на базовый класс
- **RAII через `std::unique_ptr`** — автоматическое управление памятью без ручных `new` / `delete`
- **Разделение интерфейса и реализации** — объявления в заголовочных файлах, определения в `.cpp`

## Архитектура классов

```
Character (абстрактный)
│   name, health, damage
│   virtual attack() = 0
│   virtual describe() = 0
│   virtual takeDamage()
│   isAlive(), getName()
│   virtual ~Character()
│
├── Hero
│       реализует attack(), describe()
│
└── Monster (абстрактный)
    │   expReward
    │   getExpReward()
    │
    ├── Goblin
    │       реализует attack(), describe()
    │
    └── Dragon
            реализует attack(), describe()
```

**Боевая система** — свободная функция `battle(Character*, Character*)` принимает двух любых бойцов и проводит пошаговый бой до победы одного из них. Функция не зависит от конкретных типов: имена, урон и состояние берутся из объектов через виртуальные вызовы и геттеры. Один и тот же код корректно проводит бой между любой парой существ.

Участники хранятся в `std::vector<std::unique_ptr<Character>>` — память освобождается автоматически при уничтожении вектора, корректную очистку каждого наследника обеспечивает виртуальный деструктор.

## Сборка и запуск

**Через g++:**
```bash
g++ -std=c++17 -Iinclude src/*.cpp -o rpg
./rpg
```

**Через CMake:**
```bash
cmake -B build
cmake --build build
```

## Пример боя

```
Раунд 1
Артём атакует Goblin
Goblin атакует Артём
Артём 92 20
Goblin 10 8 20
Раунд 2
Артём атакует Goblin
Победил Артём!
```

Тот же код для более сильного противника даёт другой исход:

```
Раунд 3
Кашед атакует Dragon
Dragon атакует Кашед
Победил Dragon!
```

Исход боя определяется характеристиками существ, а не логикой функции — в этом и проявляется полиморфизм.
